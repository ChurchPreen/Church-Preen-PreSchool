## 🔧 How easy is it for others to make changes?

### The admin panel (for committee members — NO coding needed)

Once the site is live on GitHub Pages, anyone can:

1. Go to **yoursite.com/admin/**
2. Enter the password + their GitHub token
3. Use a simple dashboard to update:
   - **News posts** — add/edit/delete, with date and emoji
   - **Term dates** — just type the new dates and save
   - **Events** — add upcoming fundraisers etc
   - **Gallery** — upload photos from their phone or computer
   - **Contact info** — phone, email, address, opening hours
   - **Staff** — names, roles, bios
   - **Admissions FAQ** — edit any question or answer
   - **Ofsted** — update grades and report links
   - **Policies** — add or remove policy names
4. Click **Save** — website updates within seconds

**No coding. No technical knowledge. Works from any device, anywhere.**

The only slightly technical bit is setting up a GitHub token (one-time, takes 5 minutes, full instructions in the README).

### Bigger changes (design, new sections, layout)

For anything beyond content — new section, colour change, layout tweak — the easiest route is to **continue in Claude** (claude.ai). Upload the HTML file, describe what you want, and Claude will make the change. This is exactly how this whole site was built.

A competent web developer could also make changes directly to the HTML — it's a single self-contained file with clean, well-organised code.

### What CANNOT be changed easily without help

- The overall page structure and layout
- Adding completely new page types
- The social media feed integration (needs Meta developer setup)

These all need either Claude or a developer.
